Memory Types and Bandwidth

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Memory Types for Different Systems

  • Memory modules vary based on system use cases:
    • Desktop computers: Use standard memory.
    • Servers (web/database): Require advanced memory types for reliability.

Parity Memory

  • Includes an extra parity bit per byte (8 bits → 9 bits).
  • Detects errors but cannot correct them.
    • System may halt and require a reboot.
    • Uses even parity: Parity bit ensures total 1s in a byte are even.
    • Example parity calculations:
      • 11100111 → 6 ones (even) → parity bit = 0.
      • 00000010 → 1 one (odd) → parity bit = 1.
  • Error detection process:
    • Write: Store byte + parity bit.
    • Read: Recalculate parity and compare to stored bit.
    • Mismatch = memory error.

ECC (Error-Correcting Code) Memory

  • Detects and corrects errors.
  • Allows system to continue running without halting.
  • Physically similar to standard/parity memory but requires compatible hardware.

Memory Bandwidth

  • Measures data transfer speed between CPU and memory.
  • Units: MT/s (Million Transfers per Second).
    • Example: DDR5 32GB module @ 5,600 MT/s.
  • Bottlenecks:
    • CPU idle time waiting for memory transfers.
    • Throughput limits due to physical pathways.

Multi-Channel Memory

  • Increases bandwidth by adding parallel channels between CPU and memory.
  • Types:
    • Dual-channel: 2 modules (doubles throughput).
    • Triple/Quad-channel: 3/4 modules (proportional increase).
  • Requirements:
    • Modules must be identical (same type, make, model).
    • Motherboard support (check docs for color-coded slots).
  • Example: Two 16GB modules > one 32GB module for bandwidth.
