Cloud Computing Models

CompTIA A+ 220-1201 - Core 1 | Lesson 3.1

Introduction to Cloud Computing

  • Cloud computing is not just a hosted server—it provides extensive functionality beyond traditional hosting.
  • Key advantages:
    • Global deployment: Host applications/services anywhere in the world.
    • Scalability: Access to virtually unlimited CPU, storage, and network resources.
    • Rapid deployment: Spin up/down entire application instances instantly (vs. weeks/months for physical servers).
    • Cost efficiency: Pay-as-you-go model eliminates upfront infrastructure costs.
    • Geographical flexibility: Deploy apps in regions closest to users (e.g., US, Europe).
    • Elasticity: Scale resources up/down based on demand (e.g., seasonal workloads).

Cloud Deployment Models

  • Public Cloud:
    • Hosted by third-party providers (e.g., Microsoft Azure, Amazon AWS, Rackspace).
    • Accessible via the public internet.
    • Shared infrastructure with multi-tenancy.
  • Private Cloud:
    • Dedicated cloud infrastructure within an organization’s private data center.
    • Full control over security, data, and compliance.
    • Requires upfront capital for hardware/storage.
  • Hybrid Cloud:
    • Combines public and private clouds.
    • Example: Use private cloud for internal apps and public cloud for customer-facing services.
  • Community Cloud:
    • Shared infrastructure among organizations with common needs (e.g., healthcare, government).
    • Reduces costs while maintaining control.

Cloud Service Models

  • Infrastructure as a Service (IaaS):
    • Aka Hardware as a Service (HaaS).
    • Rent raw compute resources (CPU, storage, networking).
    • Customer responsibilities:
      • Install/maintain OS, applications, and security.
      • Manage data and backups.
    • Provider responsibilities:
      • Physical hardware, data center, and network connectivity.
    • Example: Hosting a web server (e.g., professormesser.com).
    • Pros: High control/flexibility. Cons: High management overhead.
  • Platform as a Service (PaaS):
    • Provides a development platform (OS, middleware, runtime).
    • Customer responsibilities:
      • Develop/deploy custom applications.
      • Manage application data and user access.
    • Provider responsibilities:
      • Underlying infrastructure (servers, storage, networking).
      • Platform updates/maintenance.
    • Example: Salesforce.com (custom app development).
    • Analogy: "Building blocks" for assembling applications without managing infrastructure.
  • Software as a Service (SaaS):
    • On-demand software accessed via web browser.
    • Customer responsibilities:
      • User authentication (e.g., usernames/passwords).
      • Client devices (PCs, tablets, phones).
    • Provider responsibilities:
      • Everything from infrastructure to application updates.
      • Data storage and security.
    • Examples: Google Mail, Microsoft 365.
    • Pros: Zero maintenance, no development needed. Cons: Limited customization.

Responsibility Matrix

  • Defines shared responsibilities between provider and customer.
  • On-Premises:
    • Customer manages everything (data, apps, OS, hardware, networking).
  • IaaS:
    • Provider: Data center, hardware, networking.
    • Customer: OS, apps, data, security.
  • PaaS:
    • Provider: Platform (OS, runtime, middleware).
    • Customer: Applications, data.
  • SaaS:
    • Provider: Entire stack (app, data, infrastructure).
    • Customer: User access, client devices, identity management.
