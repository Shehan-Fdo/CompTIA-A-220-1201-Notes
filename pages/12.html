Local Area Networks (LANs) and Virtual LANs (VLANs)

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Introduction to LANs

  • A Local Area Network (LAN) is a group of devices connected into a single broadcast domain.
  • When a broadcast is sent from one device, all devices in the same LAN receive it.
  • Separate physical switches create separate broadcast domains, isolating broadcasts between them.
  • Example: Devices on a red network (Switch A) cannot see broadcasts from a blue network (Switch B).

Challenges with Physical LANs

  • Inefficient use of resources:
    • Underutilized switches (e.g., 24-port switch with only 2 devices connected).
    • Requires additional physical space and power sources for multiple switches.
  • Solution: Consolidate devices onto a single physical switch while maintaining logical separation.

Virtual LANs (VLANs)

  • A VLAN is a logical subdivision of a physical switch into multiple broadcast domains.
  • Key Benefits:
    • Cost-efficient: Uses a single switch instead of multiple physical switches.
    • Space-saving: Reduces rack space and power consumption.
    • Security: Isolates traffic between VLANs (e.g., red VLAN vs. blue VLAN).
    • Flexibility: Devices on the same switch can belong to different VLANs.
  • How VLANs Work:
    • Interfaces on a switch are assigned to specific VLANs (e.g., VLAN 1 for "Gate Room," VLAN 2 for "Dialing Room").
    • Broadcasts are confined to their VLAN (e.g., a broadcast on VLAN 1 is invisible to VLAN 2).

Inter-VLAN Communication

  • By default, devices on different VLANs cannot communicate directly.
  • Routing Required:
    • A router (or Layer 3 switch) is needed to route traffic between VLANs.
    • Some switches support internal routing (e.g., inter-VLAN routing).
    • External routers can also be used for this purpose.

Virtual Private Networks (VPNs)

  • A VPN encrypts data sent over a network, ensuring confidentiality and security.
  • VPN Concentrator:
    • A device (hardware/software) that encrypts/decrypts VPN traffic in real-time.
    • Often integrated into firewalls or dedicated appliances.
  • Types of VPNs:
    • Client-to-Site VPN:
      • Used by remote users (e.g., working from home) to connect to a corporate network.
      • Traffic is encrypted between the client and the concentrator.
      • Can be configured as always-on (auto-connects on login).
    • Site-to-Site VPN:
      • Connects entire networks (e.g., corporate HQ to remote offices) over the internet.
      • Uses firewalls as VPN concentrators to encrypt traffic between sites.
      • Ensures secure communication between physically separate locations.
  • VPN Implementation:
    • Hardware-based (e.g., firewall appliances).
    • Software-based (e.g., VPN server on an existing machine).
    • Built-in client support in most operating systems (e.g., Windows, macOS).
    • Third-party VPN software (e.g., OpenVPN, Cisco AnyConnect).

Key Takeaways

  • LANs = Single broadcast domain; VLANs = Logical separation within a switch.
  • VLANs improve efficiency, security, and scalability.
  • Inter-VLAN communication requires a router or Layer 3 switch.
  • VPNs provide secure remote access (client-to-site) and site connectivity (site-to-site).
  • VPN concentrators handle encryption/decryption, often via firewalls.
