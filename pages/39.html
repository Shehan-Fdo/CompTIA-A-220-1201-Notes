BIOS and UEFI Configuration

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Accessing BIOS/UEFI

  • Access during boot by pressing a function key (commonly Del, F1, F2, or Ctrl+Alt+S).
  • For virtual machines:
    • Hyper-V (Windows 10/11), VMware Workstation (Windows/macOS), or VMware Fusion (macOS) support BIOS access.
    • VirtualBox does not support virtual BIOS configuration.
    • Alternative: Use a UEFI BIOS simulator (search online).
  • Fast Startup (Windows 10/11) prevents BIOS access on shutdown. Solutions:
    • Hold Shift while clicking Restart.
    • Navigate to Settings > Update & Security > Recovery > Advanced Startup > Restart Now.
    • Use msconfig (System Configuration utility).
    • Interrupt boot 3 times to force a full restart on the 4th attempt.

BIOS/UEFI Warnings and Best Practices

  • Risks: Incorrect changes can cause boot failures or system instability.
  • Best Practices:
    • Document changes (write down or take screenshots).
    • Understand settings before modifying (especially CPU/memory configurations).
    • Backup BIOS configuration before making changes.

Key BIOS/UEFI Features

  • Hardware Control:
    • Enable/disable hardware (e.g., USB ports, integrated peripherals).
    • Disabled hardware is invisible to the OS.
  • Boot Order:
    • Prioritize boot devices (e.g., SSD, USB, Network, M.2).
    • Move devices up/down in the sequence via BIOS menu (e.g., Startup > Primary Boot Sequence).
  • USB Configuration:
    • Enable/disable USB ports (security measure).
    • Example: DoD banned USB drives (2008) due to Agent.btz worm.
    • Settings under Devices > USB Setup (enable/disable ports, adjust USB legacy support).
  • Fan Control:
    • Monitor and adjust fan speeds based on temperature.
    • Modes:
      • Best Performance: Max cooling (loud).
      • Best Experience: Balanced (quiet).
      • Full Speed: Always max RPM.
    • Configure under Power > Intelligent Cooling.
  • Secure Boot (UEFI only):
    • Prevents malware from loading before the OS.
    • Uses digital signatures to verify:
      • OS integrity.
      • BIOS firmware.
      • Bootloader.
    • Disable for legacy OS (e.g., Windows 7).
    • Configure under Security > Secure Boot (manage keys, enable/disable).
  • Password Protection:
    • User Password (Boot Password): Required to boot the system.
    • Supervisor Password (BIOS Password): Required to access/modify BIOS settings.
    • Reset via motherboard jumper (e.g., CLRTC) if forgotten.

Temperature and Hardware Monitoring

  • BIOS displays real-time temperatures (CPU, memory, motherboard).
  • Useful for:
    • Diagnosing overheating.
    • Verifying cooling after hardware upgrades.
  • Access via hardware monitoring section (varies by manufacturer).

Virtualization Support

  • Enable hardware-assisted virtualization for better performance:
    • Intel VT-x (Intel CPUs).
    • AMD-V (AMD CPUs).
  • Configure under Advanced > CPU Setup (e.g., AMD Secure Virtual Machine).
  • Required for hypervisors (e.g., Hyper-V, VMware).

BIOS/UEFI Storage and Reset

  • Firmware Storage:
    • BIOS/UEFI firmware stored in flash memory (upgradable).
    • Configuration stored in non-volatile flash memory (no battery dependency).
    • Legacy systems used CMOS (volatile memory + battery).
  • Resetting BIOS:
    • Physical access required (e.g., jumper on motherboard).
    • Example: CLRTC (Clear Real-Time Clock RAM) on ASUS boards.
    • Steps:
      • Power off.
      • Move jumper to clear pins (short circuit).
      • Power on to reset.
      • Return jumper to original position.
