RAID Configurations

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Introduction to RAID

  • RAID (Redundant Array of Independent Disks) provides redundancy to prevent data loss from drive failures.
  • RAID is not a backup solution—maintain separate backups.
  • Common RAID levels: RAID 0, RAID 1, RAID 5, RAID 6, and RAID 10.

RAID 0 (Striping)

  • Splits data across ≥2 drives for performance (no redundancy).
  • Example: A file split into blocks (1A, 2A, 3A) distributed alternately across drives.
  • Pros: Faster read/write speeds (parallel operations).
  • Cons: Zero redundancy—failure of any drive loses all data.

RAID 1 (Mirroring)

  • Creates an exact copy of data on ≥2 drives.
  • Pros: Full redundancy—survives single drive failure.
  • Cons: 50% storage efficiency (e.g., 2x 1TB drives = 1TB usable).
  • Replace failed drive ASAP to restore redundancy.

RAID 5 (Striping with Parity)

  • Combines striping (like RAID 0) with distributed parity for redundancy.
  • Requires ≥3 drives; 1 drive’s capacity used for parity.
  • Pros:
    • Survives single drive failure.
    • Better storage efficiency than RAID 1 (e.g., 4x 1TB drives = 3TB usable).
  • Cons:
    • Parity calculation adds CPU overhead.
    • Performance degrades during rebuild.

RAID 6 (Striping with Dual Parity)

  • Extends RAID 5 with two parity blocks for higher fault tolerance.
  • Requires ≥4 drives; 2 drives’ capacity used for parity.
  • Pros:
    • Survives two simultaneous drive failures.
    • Ideal for critical data with higher risk of multiple failures.
  • Cons:
    • Higher storage overhead (e.g., 4x 1TB drives = 2TB usable).
    • Slower writes due to dual parity calculations.

RAID 10 (Nested: Mirror + Stripe)

  • Combines RAID 1 (mirroring) and RAID 0 (striping).
  • Requires ≥4 drives (pairs of mirrored drives, then striped).
  • Pros:
    • High performance (striping) + redundancy (mirroring).
    • Survives multiple drive failures (one per mirror pair).
  • Cons:
    • 50% storage efficiency (e.g., 4x 1TB drives = 2TB usable).
    • More expensive (requires even number of drives).

Key Comparisons

  • Redundancy:
    • RAID 0: None.
    • RAID 1/10: Full mirroring.
    • RAID 5/6: Parity-based (1 or 2 drive tolerance).
  • Performance:
    • Fastest: RAID 0 → RAID 10 → RAID 5 → RAID 6.
    • Slower writes in parity RAIDs (RAID 5/6) due to calculations.
  • Use Cases:
    • RAID 0: Non-critical, speed-focused (e.g., gaming, caching).
    • RAID 1: Critical single-drive redundancy (e.g., OS drives).
    • RAID 5/6: Balanced storage/efficiency (e.g., file servers).
    • RAID 10: High-performance + redundancy (e.g., databases).
