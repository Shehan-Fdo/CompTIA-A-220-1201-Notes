POST, BIOS, and Troubleshooting

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Power-On Self-Test (POST)

  • POST (Power-On Self-Test) runs when a computer boots to verify core hardware components.
  • Checks include:
    • CPU installation and communication with BIOS.
    • Memory (RAM) presence and functionality.
    • Video subsystem (GPU/integrated graphics).
    • Basic storage and input/output controllers.
  • If POST fails, the system generates:
    • Beep codes (audible alerts via motherboard speaker).
    • Error messages (if video output is functional).
  • Beep codes vary by manufacturer (e.g., AMI, Award, Phoenix). Do not memorize them; refer to documentation.

BIOS/UEFI Basics

  • BIOS (Basic Input/Output System) or UEFI (Unified Extensible Firmware Interface) initializes hardware before handing control to the OS.
  • Key functions:
    • Manages boot order (e.g., SSD, USB, CD-ROM).
    • Maintains hardware settings (e.g., CPU, memory timings).
    • Stores date/time via the CMOS battery.
  • Common BIOS/UEFI issues:
    • Incorrect date/time: Replace the CR2032 CMOS battery.
    • Boot failures: Check boot order or reset to defaults.
    • Misconfigurations: Use Load Optimized Defaults or clear CMOS via jumper.

Troubleshooting POST/BIOS Errors

  • No video output (black screen) + beeps:
    • Likely causes: GPU failure, RAM failure, or CPU failure.
    • Steps:
      • Reseat RAM, GPU, and CPU.
      • Test with known-good components.
      • Check for loose cables (power, data).
  • No video but fans spin:
    • Possible POST failure (motherboard or PSU issue).
    • Use a multimeter to test PSU voltages (+3.3V, +5V, +12V).
  • Continuous reboots or power-offs:
    • Check for overheating (clean fans/heatsinks, reapply thermal paste).
    • Test PSU and motherboard for faults.

Windows Stop Errors (BSOD)

  • Blue Screen of Death (BSOD) indicates a critical OS failure.
  • Common causes:
    • Hardware failures (RAM, GPU, storage).
    • Driver conflicts (outdated/corrupt drivers).
    • Software issues (malware, OS corruption).
  • Key troubleshooting steps:
    • Note the stop code (e.g., 0x00000135) and search Microsoft’s database.
    • Boot into Safe Mode to isolate the issue.
    • Use Last Known Good Configuration or System Restore.
    • Roll back or update drivers.
    • Run hardware diagnostics (e.g., memtest86, manufacturer tools).
  • Tools for analysis:
    • Event Viewer (eventvwr.msc).
    • Reliability Monitor (tracks crashes/hangs over time).

Application Errors

  • Symptoms:
    • Applications crash or freeze.
    • Error messages (e.g., Error 47829 [22573]).
    • Silent failures (application disappears without warning).
  • Troubleshooting steps:
    • Check Event Viewer for detailed logs.
    • Use Reliability Monitor to identify patterns.
    • Reinstall the application (ensures clean installation).
    • Update the application/OS.
    • Test in Safe Mode to rule out conflicts.
  • Documentation:
    • Take screenshots of error messages.
    • Provide logs to the software vendor for support.

Hardware Failures

  • Overheating:
    • Symptoms: random shutoffs, throttling, fans at max speed.
    • Causes:
      • Dust-clogged fans/heatsinks.
      • Failed thermal paste.
      • Obstructed airflow.
    • Tools:
      • HWMonitor (monitors temps/voltages).
      • SpeedFan (fan control).
  • Unusual noises:
    • Grinding/scraping: Likely HDD failure (back up data immediately).
    • Clicking: Fan obstruction or failing HDD.
    • Popping/smoke: Capacitor failure (replace motherboard/PSU).
    • Rattling: Loose heat sink or component.
  • Power issues:
    • Test PSU with a multimeter.
    • Check wall outlet and power cables.
    • Inspect for bulging/leaking capacitors on the motherboard.

Performance Issues

  • Common causes:
    • High CPU/RAM usage (check Task Manager).
    • Low disk space (clean up or expand storage).
    • Fragmented HDD (run defrag).
    • Malware (scan with Windows Defender or Malwarebytes).
    • Power-saving modes (disable throttling in Power Options).
    • Outdated drivers/OS (run Windows Update).
  • Diagnostic tools:
    • Task Manager (real-time resource monitoring).
    • Resource Monitor (resmon).
    • Performance tab in Task Manager (historical usage).

Display Issues

  • Black screen troubleshooting:
    • Check monitor connections (HDMI, DisplayPort, VGA).
    • Verify input source on the monitor.
    • Adjust brightness/contrast settings.
    • Test with a known-good monitor/cable.
    • Boot into VGA mode (press F8 during boot) to rule out GPU driver issues.
  • Flickering/artifacts:
    • Update GPU drivers.
    • Test with a different cable/monitor.
    • Reseat the GPU (if discrete).
