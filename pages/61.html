Network Troubleshooting Steps

CompTIA A+ 220-1201 - Core 1 | Lesson X.X

Basic Connectivity Checks

  • Check for link light on Ethernet connection:
    • If present: Basic communication is established.
    • If absent: Possible bad cable or poor connection.
  • Ping the loopback address (127.0.0.1):
    • Verifies if the IP stack is functioning.
    • No response indicates an internal network configuration issue.
  • Ping the local IP address:
    • Confirms link light and network connectivity.
    • IP can be assigned via DHCP or manually configured.

Local Network Troubleshooting

  • Ping the default gateway:
    • Verifies communication with another device on the local network.
  • Check for APIPA address (169.254.x.x):
    • Indicates DHCP failure.
    • Results in limited or no connectivity.

External Network Troubleshooting

  • Ping an external IP address (e.g., 8.8.8.8, 9.9.9.9):
    • Confirms connectivity beyond the local network.
  • Use traceroute to identify where connectivity fails.
  • Check for firewall, ACLs, or filtering issues.

Wireless Network Issues

  • Check for interference from devices like:
    • Fluorescent lights
    • Microwave ovens
    • Cordless phones
  • Improve signal strength:
    • Move closer to the access point.
    • Change antennas or use external antennas.
    • Reposition the access point to a centralized location.
  • Adjust channel/frequency:
    • Use auto-selection or manually set to avoid interference.
  • Check signal-to-noise ratio (SNR):
    • Higher SNR = better performance.
    • Low SNR may indicate excessive interference.
  • Address multipath interference:
    • Caused by signal reflections off surfaces.

Performance Issues

  • Check for high latency:
    • Measure delays between request and response.
    • Use ping to evaluate response times.
  • Run a speed test:
    • Identifies slow links or bandwidth issues.
  • Monitor network utilization and errors.
  • Check for CPU/memory bottlenecks on local devices.
  • Use packet capture to analyze traffic patterns.

Real-Time Communication Issues

  • Monitor jitter:
    • Measures variability in packet delay.
    • High jitter causes choppy audio/video.
  • Ensure low latency for real-time applications (e.g., VoIP, video calls).
  • Check for network congestion or errors.

Intermittent Issues

  • Use continuous ping to monitor connectivity.
  • Run periodic traceroute or speed tests.
  • Check service level agreements (SLAs) for third-party issues.
  • Gather logs and packet captures during problem occurrences.

Physical Layer Issues

  • Port flapping:
    • Link light fluctuates (up/down repeatedly).
    • Possible causes: bad cable, faulty port.
    • Test by replacing cable or switching ports.

Authentication Issues

  • Check for permission denied errors in packet captures.
  • Verify credentials (username/password).
  • Ensure authentication hasn’t timed out.
