Dynamic Host Configuration Protocol (DHCP)

CompTIA A+ 220-1201 - Core 1 | Lesson 1.5

Introduction to DHCP

  • DHCP (Dynamic Host Configuration Protocol) automates IP address assignment, eliminating manual configuration.
  • Created in 1997 to manage growing networks with thousands of devices.
  • Replaced static IP assignment, which was impractical for large-scale networks.
  • Enables devices to automatically receive:
    • IP address
    • Subnet mask
    • Default gateway
    • DNS servers
    • Other IP parameters (e.g., VoIP servers)
  • Critical for modern networks (e.g., coffee shops, offices) where devices connect/disconnect frequently.

DHCP Process: DORA

  • The DORA process describes the 4-step interaction between a client and DHCP server:
  • Discover:
    • Client broadcasts a DHCP Discover message (source: 0.0.0.0:68, destination: 255.255.255.255:67).
    • Sent via UDP to locate available DHCP servers.
    • Switches forward the broadcast to all devices on the local subnet.
  • Offer:
    • DHCP server responds with a DHCP Offer (broadcast to 255.255.255.255:68).
    • Includes proposed IP address and other configuration parameters.
    • Client may receive multiple offers if multiple DHCP servers exist.
  • Request:
    • Client selects an offer and broadcasts a DHCP Request (source: 0.0.0.0:68, destination: 255.255.255.255:67).
    • Informs the server of the accepted IP address.
  • Acknowledge (ACK):
    • Server sends a DHCP ACK (broadcast to 255.255.255.255:68) to confirm the lease.
    • Client configures the assigned IP address (e.g., 10.10.10.42).
    • Lease duration prevents IP conflicts during the lease period.

DHCP Scope and Configuration

  • A DHCP scope defines the pool of IP addresses and settings for a subnet.
  • Key components of a scope:
    • Address Pool: Range of assignable IPs (e.g., 192.168.1.2–192.168.1.254).
    • Exclusions: Reserved IPs for static devices (e.g., routers, switches).
    • Subnet Mask: Defines the network segment (e.g., 255.255.255.0).
    • Lease Duration: Time an IP is reserved for a client (configurable).
    • Options: Additional parameters like DNS servers, default gateway, or VoIP settings.
  • Scopes can span multiple subnets (e.g., 192.168.1.0/24, 192.168.2.0/24).
  • Administrators can create non-contiguous pools with exclusions for static assignments.

DHCP Reservations

  • A DHCP reservation ensures a specific device always receives the same IP.
  • Use cases:
    • Servers (file/web servers).
    • Routers or critical network devices.
  • Configured via the device’s MAC address in the DHCP server.
  • Advantages over static IPs:
    • Centralized management (changes propagate automatically).
    • No manual updates required on individual devices.
  • Example:
    • Device Prometheus (MAC: XX:XX:XX) → Always assigned 192.168.1.6.
    • Device Odyssey (MAC: YY:YY:YY) → Always assigned 192.168.1.9.
  • Also called static DHCP or IP reservation.

Practical Example: Windows Server DHCP

  • DHCP service managed via Server Manager in Windows Server.
  • Scope configuration includes:
    • Scope name (e.g., 165.245.44.0).
    • Address pool (assignable IPs).
    • Active leases (view assigned IPs).
    • Reservations (MAC-to-IP mappings).
    • Additional options (e.g., DNS, gateway).
  • Router example:
    • Pool: 192.168.1.2–192.168.1.254.
    • Reservations: 192.168.1.6 (Prometheus), 192.168.1.9 (Odyssey).
