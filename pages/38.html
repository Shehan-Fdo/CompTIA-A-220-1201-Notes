BIOS and UEFI Fundamentals

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Introduction to BIOS

  • The BIOS (Basic Input/Output System) is firmware that initializes hardware during startup.
  • Also called system firmware, ROM BIOS (historically stored in Read-Only Memory), or motherboard firmware.
  • Modern systems store BIOS in flash memory (rewritable) on the motherboard, often with dual chips (main + backup) for redundancy during updates.
  • Primary role: Prepare hardware for the operating system (OS) load via the Power-On Self-Test (POST).

Power-On Self-Test (POST)

  • Diagnostic process that runs immediately at startup to verify critical hardware:
    • Checks for CPU, memory (RAM), keyboard, and mouse.
    • Displays errors on-screen if hardware fails initialization (e.g., missing RAM, no keyboard).
  • Duration: Typically a few seconds.
  • After POST, the system either:
    • Loads the OS directly, or
    • Displays a bootloader prompt (if multiple OS options exist).

Legacy BIOS vs. UEFI

  • Legacy BIOS:
    • Text-based interface (no mouse support; uses keyboard keys like F1–F12, arrow keys, or Space).
    • Over 25 years old; lacks support for modern hardware (e.g., NVMe SSDs, secure boot).
    • Limited to 16-bit processor mode and 2TB boot drives (MBR partition table).
    • Options displayed at the bottom of the screen (contextual help).
  • UEFI (Unified Extensible Firmware Interface):
    • Modern standard created by Intel; replaces legacy BIOS on all current systems.
    • Features:
      • Graphical UI with mouse support.
      • Supports 64-bit mode, GPT partition tables (drives >2TB), and secure boot.
      • Cross-manufacturer consistency (same options/layouts across brands).
      • Faster boot times via parallel hardware initialization.

UEFI BIOS Settings

  • Key configuration categories:
    • System Overview: CPU model, BIOS version, installed memory.
    • Device Management: Storage (SATA/NVMe), audio, network adapters.
    • Advanced CPU Settings: Virtualization (VT-x/AMD-V), hyperthreading.
    • Power Options: Sleep states, wake-on-LAN.
    • Security: Secure Boot, TPM (Trusted Platform Module), admin passwords.
    • Boot Options: Boot order, CSM (Compatibility Support Module for legacy OS).
  • Warning: Incorrect changes can cause system instability or boot failures. Document changes and create backups before modifying.

BIOS/UEFI Updates

  • Updates provide:
    • Support for new hardware (CPUs, GPUs, storage).
    • Security patches (e.g., Spectre/Meltdown mitigations).
    • Bug fixes (e.g., USB compatibility, fan control).
  • Risks:
    • Failed updates can brick the motherboard (why backup chips exist).
    • Always use manufacturer-provided tools (e.g., HP BIOS Update, ASUS EZ Flash).
  • Best Practices:
    • Backup current BIOS version.
    • Use uninterruptible power (avoid updates during storms/outages).
    • Follow vendor documentation exactly.
