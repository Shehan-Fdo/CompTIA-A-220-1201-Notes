Encryption and Security Modules

CompTIA A+ 220-1201 - Core 1 | Lesson 3.7

Introduction to Encryption

  • Encryption secures data by making it unreadable to unauthorized users.
  • Used in storage (hard drives, SSDs), transmission (networks, mobile signals), and authentication.
  • Encryption/decryption processes are often publicly documented (open standards).
  • Security relies on keys (digital), similar to physical keys for locks.
  • Challenge: Protecting the encryption keys themselves.

Trusted Platform Module (TPM)

  • Purpose: Hardware-based security for encryption keys on single devices.
  • Components:
    • Cryptographic processor: Handles encryption/decryption and random number generation.
    • Persistent memory: Stores burned-in keys (unique per device).
    • Volatile memory: Temporary storage for keys/data.
    • Security features: Password protection, anti-tampering.
  • Key Features:
    • Unique, device-specific root key (never leaves the TPM).
    • Enables full-disk encryption (e.g., BitLocker).
    • Prevents data access if storage is moved to another device.
    • Acts as a root of trust for hardware authentication.
    • Detects unauthorized changes to system hardware/software.
  • BIOS Configuration:
    • Enable/disable TPM under Security > TCG (Trusted Computing Group).
    • Options to clear TPM (reset keys) or configure data deletion.
  • Use Cases:
    • Mobile device screen locks.
    • Full-disk encryption (e.g., laptops, desktops).
    • Secure boot processes.

Hardware Security Module (HSM)

  • Purpose: Centralized, high-security key management for enterprise/data centers.
  • Key Features:
    • Stores and backs up keys for multiple devices.
    • Cryptographic acceleration: Offloads encryption/decryption from servers.
    • High-end hardware with tamper-resistant design.
    • Supports remote key management.
  • Types:
    • Enterprise HSMs: Deployed in data centers (e.g., for web server keys, certificate authorities).
    • Personal HSMs: Portable devices (e.g., for cryptocurrency).
  • Comparison with TPM:
    • TPM: Single-device, built into motherboard, local encryption (e.g., BitLocker).
    • HSM: Multi-device, centralized, high-performance, enterprise-scale.

Practical Applications

  • Full-Disk Encryption:
    • TPM stores keys for BitLocker/similar tools.
    • Data remains encrypted if drive is removed.
  • Network Security:
    • TPM verifies device identity (prevents spoofing).
    • HSM manages keys for TLS/SSL certificates.
  • Data Center Security:
    • HSMs protect keys for web servers, databases, and certificate authorities.
    • Accelerates encryption for high-traffic systems.
