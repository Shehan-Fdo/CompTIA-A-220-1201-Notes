Mobile Device Management

CompTIA A+ 220-1201 - Core 1 | Lesson 1.3

MDM (Mobile Device Management)

  • MDM is a system used to manage company-owned and user-owned mobile devices.
  • It addresses policies for BYOD (Bring Your Own Device).
  • Provides centralized management of all mobile devices from a single console.
  • Allows administrators to set policies on apps, data, cameras, and other device features.
  • Can control the entire device or create a separate, secure "partition" for corporate data, leaving personal data untouched.
  • Manages access control by enforcing security policies like screen locks and PINs.

BYOD (Bring Your Own Device)

  • BYOD, or Bring Your Own Technology, is a policy where employees use their personal devices for work purposes.
  • The employee owns the device but must meet the company's security and usage requirements.
  • Security Challenges:
    • The device is used for both personal and work activities, creating a security risk.
    • Raises concerns about how corporate data is protected on a personal device.
    • Poses a risk of data leakage if the device is sold, traded in, or lost without being properly wiped.

COPE (Corporate Owned, Personally Enabled)

  • COPE is a model where the company purchases the device and issues it to the employee.
  • The device is used for both corporate and personal activities, but the organization maintains full control.
  • This is similar to how companies manage corporate laptops and desktops.
  • All information on the device is protected by corporate policies, and the company can delete the data at any time.
  • A variation is CYOD (Choose Your Own Device), where the employee can select a device from a company-approved list.

MDM Policy Enforcement

  • Corporate Email Configuration:
    • The MDM can automatically push email configurations (account details, server address, etc.) to devices.
    • The user does not need to configure anything manually.
  • Two-Factor Authentication:
    • MDM can enforce the use of specific two-factor authentication types, such as biometrics or authentication apps.
  • Corporate Applications:
    • Policies can allow or restrict the installation of certain apps.
    • Helps prevent unauthorized application usage and ensures only approved software is on the device.

Mobile Device Synchronization

  • Many settings like telephone and text messaging are preconfigured.
  • Email configurations often vary and need to be set up, which can be automated by MDM.
  • Data Synchronization is crucial for backup and recovery.
    • Synchronization settings can be based on data rates, speeds, and network types.
  • Data Caps and Transfer Costs:
    • Syncing over cellular networks can have significant financial costs.
    • Policies can be set to synchronize only over 802.11 (Wi-Fi) to avoid cellular data charges.
    • It's important to verify data restrictions and costs with the wireless carrier.
  • Synchronization settings can control which data types are synced (e.g., Calendar, Contacts, Photos, Mail).

Business Applications

  • Configuration for business apps like Mail and cloud storage is a common task.
  • Mobile operating systems have built-in support for many different services (iCloud, Microsoft Exchange, Google, etc.).
  • Authentication is required, usually with a username and password.
  • After authenticating, you can select which specific items you want to synchronize (e.g., Mail, Contacts, Calendars, Notes).
  • The setup process is generally the same for different services and is integrated into the mobile device's settings.
