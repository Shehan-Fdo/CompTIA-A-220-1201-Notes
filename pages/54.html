Virtualization Fundamentals

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Hypervisor Overview

  • Hypervisor (or Virtual Machine Manager (VMM)) manages interaction between virtual machines (VMs) and physical hardware.
  • Allocates resources: CPU cycles, memory, networking, and storage.
  • Modern hypervisors leverage CPUs with built-in virtualization support (Intel VT or AMD-V).
  • Each VM runs a guest OS and applications, requiring significant overhead (CPU, RAM, storage).

Types of Hypervisors

  • Type 1 (Bare Metal):
    • Runs directly on hardware (no host OS).
    • Examples: VMware ESXi, Microsoft Hyper-V.
    • Used in enterprise/data centers for performance.
  • Type 2 (Hosted):
    • Runs on top of a host OS (e.g., Windows, macOS).
    • Examples: VMware Workstation, VirtualBox, Parallels Desktop.
    • Used for desktop virtualization or testing.

Resource Allocation

  • Memory:
    • Hypervisor allocates physical RAM to VMs.
    • Ensure sufficient RAM for concurrent VMs.
  • Storage:
    • Each VM requires space for guest OS, applications, and data.
    • Use fast storage (e.g., SSD) for better performance.
  • Networking:
    • Hypervisor controls VM network access via:
      • Shared NAT: VMs share a private subnet; hypervisor performs NAT for external communication.
      • Bridged: VMs appear as independent devices on the local network (same subnet).
      • Private: VMs communicate only with each other (isolated).

Security Considerations

  • VM Escape:
    • Malware exploits hypervisor flaws to break out of a VM and access other VMs/host.
    • No major breaches reported, but a critical risk.
  • VM Hardening:
    • Treat each VM as a standalone system:
      • Install firewalls and anti-malware.
      • Apply OS hardening (patches, least privilege).
  • Malicious VMs:
    • Avoid downloading pre-built VMs from untrusted sources (may contain embedded malware).
    • Build VMs from trusted ISOs or verified templates.

Virtual Desktop Infrastructure (VDI)

  • Hosts desktops as VMs on a central server/cloud.
  • Users access via thin clients (keyboard, mouse, display).
  • Also called Desktop as a Service (DaaS).
  • Benefits:
    • Reduces local hardware requirements.
    • Centralized management and security.
    • Leverages high-bandwidth networks for remote display.

Application Containerization

  • Alternative to VMs: lightweight, portable containers.
  • Key Differences vs. VMs:
    • No guest OS per container (shares host OS kernel).
    • Smaller footprint, faster deployment.
    • Limited to host OS compatibility (e.g., Linux containers on Linux host).
  • Examples:
    • Docker: Popular containerization platform.
    • Containers isolate apps but share OS resources.
  • Use Cases:
    • Microservices, scalable apps, and CI/CD pipelines.
