TCP/UDP Port Numbers

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Introduction to Port Numbers

  • Port numbers are critical for TCP and UDP communication.
  • Essential for troubleshooting network issues and firewall configuration.
  • Memorization becomes intuitive with practical use (e.g., firewall rules, network troubleshooting).
  • Focus on:
    • The port number and associated protocol.
    • Purpose and network usage of each protocol.

File Transfer Protocol (FTP)

  • Ports: TCP 20 (data transfer), TCP 21 (control/admin).
  • Used for cross-platform file transfers (e.g., uploading/downloading files).
  • Supports:
    • Authentication (username/password or multi-factor).
    • Anonymous login (generic access).
    • File management (list, add, delete, rename).

Secure Shell (SSH) vs. Telnet

  • SSH:
    • Port: TCP 22.
    • Purpose: Encrypted remote command-line access.
    • Replaces Telnet due to security risks.
  • Telnet:
    • Port: TCP 23.
    • Risk: Transmits data in cleartext (no encryption).
    • Usage: Legacy systems only (avoid in modern networks).

Email Protocols

  • SMTP (Simple Mail Transfer Protocol):
    • Port: TCP 25.
    • Purpose: Sends emails between mail servers.
    • Also used by devices (e.g., printers, apps) to send emails.
  • POP3 (Post Office Protocol v3):
    • Port: TCP 110.
    • Purpose: Downloads emails to a single client.
  • IMAP4 (Internet Message Access Protocol v4):
    • Port: TCP 143.
    • Advantages: Syncs emails across multiple clients, supports folders/labels.

Domain Name System (DNS)

  • Port: UDP 53.
  • Purpose: Translates FQDNs (e.g., www.professormesser.com) to IP addresses.
  • Redundancy: Multiple DNS servers ensure availability.

Dynamic Host Configuration Protocol (DHCP)

  • Ports: UDP 67 (server), UDP 68 (client).
  • Purpose: Automatically assigns:
    • IP addresses.
    • Subnet masks.
    • Default gateways.
    • DNS servers.
  • Features:
    • Leases: Temporary IP assignment with renewal options.
    • Reservations: Permanent IPs for critical devices (e.g., printers, servers).

Hypertext Transfer Protocol (HTTP/HTTPS)

  • HTTP:
    • Port: TCP 80.
    • Risk: Unencrypted (data in cleartext).
  • HTTPS:
    • Port: TCP 443.
    • Security: Encrypted communication (preferred for web traffic).

Windows Protocols

  • Server Message Block (SMB)/CIFS:
    • Ports: TCP 445 (modern), TCP 139 (legacy with NetBIOS).
    • Purpose: File/printer sharing between Windows devices.
  • NetBIOS (Legacy):
    • Ports: UDP 137 (name service), TCP 139 (session setup).
    • Note: Modern Windows uses direct SMB (port 445).

Directory Services (LDAP)

  • Port: TCP 389 (LDAP), TCP 636 (LDAP-S).
  • Purpose: Accesses directory databases (e.g., Active Directory).
  • Structure: Organized by:
    • Organization (O).
    • Organizational Unit (OU).
    • Common Name (CN).
  • Use Cases: User/authentication management, resource location.

Remote Desktop Protocol (RDP)

  • Port: TCP 3389.
  • Purpose: Remote control of Windows devices.
  • Features:
    • Full desktop or single-app access.
    • Cross-platform clients (Mac, Linux, mobile).
