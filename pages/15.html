IP Address Configuration

CompTIA A+ 220-1201 - Core 1 | Lesson 1.2

Key IP Configuration Parameters

  • IP Address: Unique identifier for each device (e.g., 192.168.1.165).
  • Subnet Mask: Determines the local subnet (e.g., 255.255.255.0).
    • Not sent across the network; must be provided by the network administrator.
    • Combined with IP address to define the IP subnet.
  • Default Gateway: IP address of the local router (e.g., 192.168.1.1) for communication outside the local subnet.
  • Additional Parameters:
    • DNS servers (for name resolution).
    • NTP servers (for time synchronization).
    • VoIP servers (for voice services).

Static vs. Dynamic IP Addressing

  • Static IP Address: Manually configured and does not change.
    • Used for devices like switches, routers, firewalls, and printers.
    • Requires manual input of IP address, subnet mask, gateway, and DNS.
    • Errors in configuration can disrupt network connectivity.
  • Dynamic IP Addressing (DHCP): Automatically assigns IP addresses from a pool.
    • Scalable for large networks (e.g., corporate environments).
    • Reduces manual configuration errors.

DHCP Reservations

  • Links a device’s MAC address to a specific static IP address on the DHCP server.
  • Benefits:
    • Centralized management of IP assignments.
    • Automatic updates if gateway or DNS IP addresses change.
    • Eliminates manual reconfiguration on individual devices.
  • Process:
    • Device set to DHCP mode.
    • DHCP server assigns the reserved IP based on the MAC address.

Automatic Private IP Addressing (APIPA)

  • Used when a DHCP server is unavailable and the device is configured for DHCP.
  • Assigns a link-local IP address from the range 169.254.0.0 to 169.254.255.255.
    • Excludes 169.254.0.0–169.254.0.255 and 169.254.255.0–169.254.255.255.
    • Usable range: 169.254.1.0–169.254.254.255.
  • Limitations:
    • Allows communication only within the local subnet.
    • No internet access (cannot route outside the subnet).
  • Address Assignment Process:
    • Device sends an ARP request to check if the randomly chosen APIPA address is in use.
    • If no response, the address is assigned to the device.
  • Indicates a DHCP failure; troubleshooting required to restore DHCP connectivity.

Best Practices

  • Avoid manual static IP configuration for large networks (not scalable).
  • Use DHCP reservations for static assignments in enterprise environments.
  • APIPA addresses are a troubleshooting tool—not a permanent solution.
  • Critical infrastructure (e.g., routers, servers) should use static IPs or DHCP reservations.
